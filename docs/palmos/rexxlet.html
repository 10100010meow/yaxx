Rexxlets™

As mentioned in the introduction, Rexxlets are ordinary Rexx scripts and procedures that you can be launched anytime, anywhere, at the swipe of your pen, while not having to switch out of your currently running Palm OS application.  Combined with the expressive power of URL streams, Rexxlets are intended as both popup utilities as well as a way to glue together and hyperlink thousands of existing Palm OS applications and databases, since Rexxlets can transfer, read, and write data among applications and databases.

Rexxlets can be launched in a variety of ways.  They're launched by way of either a system extension ("hack") on Palm OS 3.x and 4.x, or system notifications on Palm OS 5.x, and allow you to assign custom pen strokes to Rexx scripts.  In the future they can be launched from a menu or from application icons.

The following section describes how to configure Rexxlets.  Insiructions involving a system extension manager, such as X-Master, only apply to Palm OS 3.x and 4.x users.  Palm OS 5.x users can simply start the Rexxlet application.  The configuration instructions are then the same for both.

It is assumed that you are familiar with what was discussed in the Rexxlet section of the download and installation instructions page and that you already have a system extension manager installed.

Enabling Rexxlets

To use Rexxlets, you will first need to make sure that the Rexxlet extension is enabled in whichever extension manager you are using.  If you are using X-Master, then the main screen will look something like this, with the check in the checkbox next to "RexxletHack 1.0" indicating that the Rexxlet extension is enabled.

Configuring Rexxlets

To configure Rexxlets, you need to use the "configure" command from whichever extension manager you are using.  If you are using X-Master, then the configure command is on a button labelled Configure in X-Master's main screen, as just shown in the Enabling Rexxlets section.  Note that Rexxlets can be configured without the Rexxlet extension being enabled, and this is perfectly legitimate.

The Rexxlet configuration screen looks like this.  

It is intended to be used by first selecting a pen stroke combination from the popup lists under the labels From and To, and then setting the Rexxlet options for that pen stroke.  As you select a different from and to pen location, the options will indicate their corresponding current settings for that particular pen stroke.  Please see the meaning of the OK, Defaults, and Cancel buttons below so that you understand: when any and all your settings are saved, how to revert them back to the factory defaults, and how to cancel and revert them back to the last saved settings. 

The Rexxlet configuration screen, titled RexxletHack Prefs by convention, contains the following items.

  • Custom Stroke
  • Is the custom stroke enabled?
  • URL
  • Args
  • Copy and paste result?
  • Pause on return?
  • OK
  • Defaults
  • Cancel

Custom Stroke

This allows you to select the custom pen stroke to associate with a Rexx script.  There are currently eight (8) custom pen strokes supported, the four (4) "soft" buttons (red areas on the PacMan screen below) labelled App, Menu, Calc, and Find, in combination with either the Graffiti area (green area) or the display area (purple area).

Is the custom stroke enabled?

This allows you selectively enable or disable certain pen strokes.  Disabling a pen stroke is not only useful if have no Rexx script assigned to it, but is also useful if you don't want it to interfere with another application that might try to use the same pen stroke or if you simply want to disable it but keep its configuration settings.

URL?

This allows you to use a URL to specify which Rexx script is associated with the pen stroke.  In addition to the dm: and file: schemes, you can also use console:in and clipboard:in as discussed here to refer to an input console (where you can enter a script) or the clipboard (where a script exists), respectively.  And, as mentioned here, examples of some of these are preconfigured.

A blank URL, or ?, will cause a dialog to pop up that will allow you to enter a URL when you use the pen stroke .

Although not yet implemented, or even fully designed, fragment identifiers on URLs can also be used in this context to denote a particular procedure within a Rexx script.  

  1. Fragment identifiers on file: URLs can possibly be useful if you want to create a single file as a repository for Rexxlets procedures, where the file would be referenced like this, file:///MyRexxlets, and the procedure within it like this, file:///MyRexxlets#qt.   

  2. Fragment identifiers on dm: URLs can possibly be useful if you want to reference scripts by some conventional naming scheme, rather than by a record number, which can be hard to use and prone to errors.  For example, if you identify each script in MemoPad by entering the first line of the script in a format like this,
         /*name*/
    then it may be possible to reference the script by name, dm:///MemoDB/Rexx/#name, rather than by number, dm:///MemoDB/Rexx/3.

Args?

This allows you to specify the source arguments for the Rexx script, i.e., Rexx's PARSE ARG.  You can specify one of the following.

  • None for no arguments.

  • Clipboard to use the contents of the clipboard at the time the Rexxlet is launched.

  • Popup to pop up a dialog that will allow you to enter the arguments at the time the Rexxlet is launched.

  • Selection to use the current selection in the current field in the currently running application at the time the Rexxlet is launched.  This is a key mechanism for hyperlinking Palm OS applications and databases.

Copy and paste result?

This allows you to specify whether or not you want to have the result that's returned from the Rexx script (Rexx's RESULT, not the contents of the Rexxlet console) automatically copied to the clipboard and then pasted into the current field in the currently running application at the time the Rexxlet is launched.  

See here for an example of how effectively this can be utilized.

Pause on return?

This allows you to specify whether or not you want to cause a pause when the Rexx script is done before returning to the application from which the Rexxlet was launched.  Pausing on return will be done in own of two (2) ways, depending on whether or not the the Rexx script performs console I/O.

  1. If the Rexx script performs console I/O—and thus causes the Rexxlet console to be displayed when the Rexx script is being run—then pausing on return will be done by way of a Return button on the Rexxlet's console that you will have to press to return to the launching application.  

  2. If the Rexx script performs no console I/O—and thus no Rexxlet console will be displayed—then pausing on return will be done by way of a separate dialog box that pops up that you will have to dismiss to return to the launching application.

Note that if the Rexxlet does not pause on return or copy and paste its results, and your Rexx script performs no console I/O, you may have no indication that the Rexxlet was launched and the Rexx script ran. 

Also note that you can choose not to pause on return, but achieve the same behavior by performing a PARSE PULL in your Rexx script just before the script returns.

OK

The OK button will save all the current settings and exit back to the system extension manager's main screen.  This is the only way to save any and all settings.  As you make changes on the form, and switch among from/to pen stroke combinations, all of your changes are remembered until you save them.  If you press Cancel, all the settings you made since the last time you saved them will be lost.

Defaults

The Defaults button will make changes to all the settings so as to revert them back to their "factory" defaults, as set when you installed the Rexxlet extension.  This button does not cause the RexxletHack Prefs dialog to exit because these defaults settings are not saved until you press OK.  If you press Cancel, all these default settings as well as any additional ones you made will be lost.

Cancel

The Cancel button will simply exit back to the system extension manager's main screen.  All the changes you made since the last time you saved them will be lost.
